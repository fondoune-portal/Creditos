🚀 Implementación de Intercomunicación FondoUne

Sistema completo para navegar entre portales y compartir datos

1. Inicializar Sesión de Usuario

Simula el inicio de sesión y guarda datos del usuario para usar en todos los portales.

¿Qué pasa? Se llama a FondouneSession.initUser() para guardar los datos del usuario en sessionStorage.

2. Guardar Solicitud de Crédito

Simula completar un formulario de solicitud. Los datos se guardan automáticamente.

¿Qué pasa? Se llama a FondouneSession.saveCreditApp() para guardar la solicitud de crédito.

3. Navegar Entre Portales

Navega a otros portales con los datos que guardaste. Los datos se recuperan automáticamente en el próximo portal.

¿Qué pasa? Se llama a FondouneNav.navigateTo() para navegar al siguiente portal.

4. Sistema de Notificaciones

Muestra notificaciones elegantes que se pueden pasar entre portales.

¿Qué pasa? Se llama a FondouneSession.showNotif() para mostrar una notificación.

Funciones Principales

Función Descripción
FondouneNav.init(portal) Inicializa el menú de navegación
FondouneNav.navigateTo(portal, data) Navega a otro portal pasando datos
FondouneSession.initUser(userData) Inicializa la sesión del usuario
FondouneSession.saveCreditApp(data) Guarda datos de la solicitud
FondouneSession.showNotif(msg, type) Muestra una notificación

6. Consejos de Implementación

✅ HACER

  • Siempre llamar a FondouneNav.init(portalKey) en cada portal
  • Usar FondouneSession.initUser() después de autenticación
  • Limpiar sesión con FondouneSession.clear() al logout
  • Verificar datos con FondouneSession.getUser() antes de usar

❌ NO HACER

  • Guardar contraseñas o datos sensibles en sessionStorage
  • Asumir que los datos existen sin verificar primero
  • Olvidar importar los scripts en el <head> de cada HTML
  • Mezclar sessionStorage con localStorage (se limpia al cerrar pestaña)
